AI Funding Radar — GPU Buyer Qualifier

2025 · Collaboration

Role: Built the qualification & enrichment pipeline (everything downstream of the classifier). A collaborator built the RSS ingestion and the initial AI-funding LLM classifier.

An n8n workflow that monitors a PRNewswire RSS feed and surfaces qualified GPU-buyer leads from the noise. The first LLM classifies whether each article describes an AI company raising a funding round — explicitly excluding IPOs, acquisitions, and VC fund formations. Articles passing this first gate trigger an instant email alert and enter the enrichment pipeline I built.

My pipeline extracts the company name via a structured-JSON LLM call, runs a Serper LinkedIn search for Machine Learning Engineers, then uses a second LLM as an evidence-based fact-checker — it reads the actual search snippets and decides whether the company genuinely employs ML talent. Companies that fail this verification are dropped. Those that pass trigger a second Serper search for decision-makers (CEO, CTO, Co-Founders, Head of AI, Head of Platform, Head of Research, Head of AI Infra, DevOps), and a final LLM call extracts each person's name, role, and LinkedIn URL as structured JSON.

Built for a GPU-database provider to identify AI startups most likely to need GPU compute right after raising capital. Uses two different OpenRouter models — a free lightweight model for the high-volume binary classification and a larger Qwen 2.5 VL model for extraction tasks — a cost-aware design choice. Defensive JSON parsing in custom code nodes handles malformed LLM output by extracting the first valid JSON object from any conversational wrapper.

II

Outlander Ventures — GTM Lead Generator

2025 · Client work

Role: Sole builder — workflow design, JavaScript enrichment, Google Sheets integration

A local-business lead generation workflow that takes a niche and location as input, pulls candidate businesses from Serper Places, and runs each prospect through an enrichment pipeline that scores buying intent. Filters at >100 reviews before any expensive HTTP work — a deliberate cost/performance choice so the workflow doesn't waste website fetches on businesses that will fail the gate anyway.

For each qualified prospect, a single JavaScript node fetches the website HTML and runs three parallel checks: it detects paid-ad investment via tracking pixels (Facebook Pixel, Google/GTM, TikTok) — a "smart money" signal that the business already spends on advertising; it extracts a contact email using a two-stage regex (preferring mailto: hrefs, falling back to raw pattern matching with junk filtering for image paths, Sentry errors, and Wix placeholders); and it flags multi-location operations by detecting common HTML patterns ("locations", "our areas", "serving these cities"). It also auto-generates a Meta Ad Library search URL per prospect — with a domain-as-fallback if the business name is missing — so the sales team can audit competitor ad creative in one click.

A subtle engineering detail: the Fetch Website node returns only HTML, breaking the data chain from Serper Places. The enrichment script handles this by reaching back to the Filter node by index and recombining the original prospect data with the scraped signals — keeping name, phone, address, rating, and reviews aligned with the new enrichment fields when results land in Google Sheets.

III

Morning Brief Bot

2026 · Personal build · Phases 1–2 shipped, Phase 3 in roadmap

Role: Sole builder — system design, n8n workflows, agentic tool design, Discord bridge, Notion API integration

A personal productivity automation I built to solve two of my own problems: not knowing what to work on first, and losing track of follow-ups. The bot reads from a Notion task database I designed (Priority, Category, Due Date, Last Touched, Today's Focus), uses Gemini to write a prioritised morning brief, and delivers it daily. Designed as a three-phase build — one-way broadcast → interactive bot → voice-call assistant — so each phase ships value without depending on the next.

Phase 1 — Daily brief (shipped). A 5-node n8n workflow runs every morning at 7:30am, queries all non-DONE tasks from Notion's 2025-09 data_sources API, then a JavaScript node sorts by priority and flags overdue items plus stale follow-ups (no contact in 3+ days). The enriched payload is sent to Gemini 2.5 Flash-Lite with a formatting prompt; the model returns a brief with top 3 priorities, overdue items, stale items, and a closing accountability question. Posted to Discord via webhook. Cost: roughly $0.0001 per run.

Phase 2 — Two-way agent (shipped). Replies in Discord now control Notion. n8n can't natively subscribe to Discord, so I wrote a small Node.js bridge running alongside n8n that uses discord.js to listen to my channel and POSTs each message to an n8n webhook. The webhook hands the message to a LangChain AI Agent backed by Gemini, with eight tools exposed to it. The agent picks tools dynamically and chains them — for "done: client proposal" it calls search_task_by_name first to retrieve the Notion page ID, then mark_task_done with that ID. Tool parameters are populated by the model itself via n8n's $fromAI(), so there's no hardcoded if/switch routing — the LLM is the router.

Phase 3 — Voice integration (roadmap). A morning phone call reads the brief aloud, asks for the day's commitment, and writes the spoken response back to Notion. Falls back to the Discord message if the call is missed.
